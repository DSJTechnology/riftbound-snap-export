# Style and Conventions
- Language: TypeScript with React functional components.
- Styling: Tailwind CSS with shadcn-ui components (Radix-based). Config files: `tailwind.config.ts`, `components.json`.
- Linting: ESLint (`npm run lint`) with React/TypeScript rules; project uses modern ES modules.
- Misc: App likely uses hooks/context; prefer consistent Tailwind utility classes and shadcn patterns. No explicit docstring/typehint conventions found beyond TypeScript typings.
